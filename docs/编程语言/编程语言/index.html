编程语言总结

我理解的编程其实就是对数据的处理。以此发散开来，那个编程其实就是变成了对下面几个元素的处理。

  1. 基础语法

    1. 数据（数据的命名，数据的生成和改变）
    2. 运算符（用来连接数据，进行运算）
    3. 控制（流程：顺序，分支，循环）
    4. 语句（数据，运算符，这些组成语句）
    5. 函数（数据，语句组成函数）

  2. 面向对象

    对象 (数据，函数组成对象)

  3. 标准库

  4. 编译器

  5. 模块（包管理）

  6. 并发编程

  7. 网络编程

用上面的模型来理解的话，那么几乎所有的编程语言都从顶端架构去理解他。这样学习某个编程语言就变成了熟悉该语言针对向上述元素的不一样表现形式而已。最后目的是得到是和某个平台的二进制文件。

程序是指令和数据的集合。

框架不是语言，但是框架是使用语言进行编程必不可少的。

回到原点#

现代机器是基于冯诺依曼机的理论建立起来的。冯诺依曼机的灵魂是存储执行。有了存储就有了衍生的各种问题，正常的是指令的顺序执行，也就是一条一条的执行。还有一种是跳转执行，调到某个地方执行。这在编程语言层面表现的就是循环，分支，函数调用等。而跳到哪里这也是个问题，在编程语言方面就表现为符号，命名空间，包等逻辑。
