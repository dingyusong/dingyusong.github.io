Extended Audio File Services#

参阅：Apple API：Extended Audio File Services

Read and write compressed files and linear PCM audio files

打开文件：

OSStatus ExtAudioFileOpenURL(CFURLRef inURL, ExtAudioFileRef _Nullable *outExtAudioFile);

持有文件：

ExtAudioFileRef
An opaque structure representing an extended audio file object.

获取文件属性：

OSStatus ExtAudioFileGetProperty(ExtAudioFileRef inExtAudioFile, ExtAudioFilePropertyID inPropertyID, UInt32 *ioPropertyDataSize, void *outPropertyData);

设置文件属性：

OSStatus ExtAudioFileSetProperty(ExtAudioFileRef inExtAudioFile, ExtAudioFilePropertyID inPropertyID, UInt32 inPropertyDataSize, const void *inPropertyData);

异步读取文件：

OSStatus ExtAudioFileRead(ExtAudioFileRef inExtAudioFile, UInt32 *ioNumberFrames, AudioBufferList *ioData);
ioNumberFrames

On input, the number of frames to read from the file. On output, the number of frames actually read. Fewer frames may be read than were requested. For example, the supplied buffers may not be large enough to accommodate the requested data. If 0 frames are returned, end-of-file was reached.

输入时，是从文件读取的帧数。 输出时，实际读取的帧数。 可能读取的帧少于请求的帧。 例如，提供的缓冲区可能不足以容纳请求的数据。 如果返回0帧，则到达文件末尾。

Disposes of an extended audio file object and closes the associated file.

处置扩展的音频文件对象并关闭关联的文件。

OSStatus ExtAudioFileDispose(ExtAudioFileRef inExtAudioFile);

AudioComponentsDescription#

identifying information for audio component.

  • componentSubType

A 4-byte code that you can use to indicate the purpose of a component. For example, you could use lpas or lowp as a mnemonic indication that an audio unit is a low-pass filter.

一个4字节的代码，可用于指示组件的用途。 例如，您可以使用lpas或lowp作为表示音频单元是低通滤波器的助记符。
