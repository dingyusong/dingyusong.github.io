The basic option is to define the target architecture. For that, use -target <triple>.     arch = x86_64, i386, arm, thumb, mips, etc.sub = for ex. on ARM: v5, v6m, v7a, v7m, etc.vendor = pc, apple, nvidia, ibm, etc.sys = none, linux, win32, darwin, cuda, etc.abi = eabi, gnu, android, macho, elf, etc.

The system name is generally the OS (linux, darwin), but could be special like the bare-metal â€œnoneâ€.

When a parameter is not important, it can be omitted, or you can choose unknown and the defaults will be used. If you choose a parameter that Clang doesnâ€™t know, like blerg, itâ€™ll ignore and assume unknown, which is not always desired, so be careful.

Finally, the ABI option is something that will pick default CPU/FPU, define the specific behaviour of your code (PCS, extensions), and also choose the correct library calls, etc.
