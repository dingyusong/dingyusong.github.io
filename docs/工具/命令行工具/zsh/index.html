ohmyzsh#

他只是一个主题，前提是需要安装zsh和git。

https://ohmyz.sh/

安装#

sh -c "$(wget https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh -O -)"

如果被墙，用这个备份。

https://gitee.com/dingyusong/ohmyzsh

sh -c "$(wget -O- https://gitee.com/dingyusong/ohmyzsh/raw/master/tools/install.sh)"

打开第一行注释让zsh生效

vi ~/.zshrc
# If you come from bash you might have to change your $PATH.
export PATH=$HOME/bin:/usr/local/bin:$PATH
# 第一行的让home下的bin在前面，如果有两套也就是用户设置的会覆盖掉系统的，比如系统自带git，但是我们又安装了一个新的git，我们安装在$HOME/bin 或者用brew安装/usr/local/bin下，这样比较符合逻辑。

ZSH_THEME_GIT_PROMPT_CLEAN="%{$fg[blue]%})"
# 右侧添加时间戳
function preexec() {
timer=${timer:-$SECONDS}
}
function precmd() {
if [ $timer ]; then
timer_show=$(($SECONDS - $timer))
if [[ $timer_show -ge $min_show_time ]]; then
RPROMPT='%{$fg_bold[red]%}(${timer_show}s)%f%{$fg_bold[white]%}[%*]%f %{$reset_color%}%'
else
RPROMPT='%{$fg_bold[white]%}[%*]%f'
fi
unset timer
fi
}
autoload -Uz add-zsh-hook
add-zsh-hook preexec preexec
add-zsh-hook precmd precmd

给Linux系统增加环境变量，是使用export命令。
为了永久性生效，则需要考虑加入到登录的profile中。 http://novell.me
这个时候要考虑你当前使用的shell,比如默认的bash shell，则可编辑用户根目录下的隐藏文件 ./bash_profile
对于zsh 而言，需要编辑 .zshrc 这个文件 vi ~/.zshrc
在最后面加上一句
export PATH=/usr/local/python-2.7.6/bin:$PATH
本文引用自http://novell.me
解释一下： 本文来自Novell迷网站 http://novell.me
环境变量中，各个值是以 冒号 分隔开的。上面的语句表示给PATH这个变量重新赋值，让它等于
/usr/local/python-2.7.6/bin
同时后面加上原来的$PATH
