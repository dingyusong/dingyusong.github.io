C++相关#

配置C++ Build

  1. 在要编译的文件 choose Terminal > Configure Default Build Task.Choose C/C++ clang++ build active file,这一步会生成一个 tasks.json

  2. 修改tasks.json args 数组添加 "-std=c++11",

  3. To run the build task that you defined in tasks.json, press ⇧⌘B or from the Terminal main menu choose Run Build Task. 这一步会生成二进制文件个符号表 dSYM文件

配置C++ Debug

至少得有一个tasks.json 并且参数配置正确，也就是完成上面的1，2两步。

注意：当前活动文件，切换到你需要调试的文件。 如果没有task.json 直接执行下面的步骤也会生成一个task.json(编译器自动生成)

  1. choose Run > Add Configuration... and then choose C++ (GDB/LLDB). 会生成一个launch.json 同时会生成调试文件启动调试界面。

VSCode: 现代工具来一套？#

参考：http://jyywiki.cn/ICS/2020/slides/5.slides#/3/4

刚拿到手，VSCode 的体验并不是非常好

  • 满屏的红线/蓝线
    • 因为 Code 并知道 NEMU 是怎么编译的
    • IDE “编译运行” 背后没有魔法

  • 另一方面，这些东西一定是可以配置的

    • 配置解析选项:

      c_cpp_properties.json
      - 解锁正确的代码解析

    • 配置构建选项:

      tasks.json
      - 解锁 make (可跟命令行参数)

    • 配置运行选项:

      launch.json
      - 解锁单步调试 (我们并不非常推荐单步调试)
