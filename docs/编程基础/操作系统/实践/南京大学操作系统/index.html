并发#

实际上的并发编程，大部分情况下：可以把任务拆成是线程之间共享的和一个线程可以独立计算的。并发的模式最常见的就是生产者和消费者模式。有一个队列，有的人不断的生产任务，有的人不断的从线程中取出任务，然后在本地执行，执行完成再把结果写到另一个地方。只要线程本地任务计算时间足够长，这样不并行的部分相对于并行的部分就很小，这样就能获得很好的并发效果。
