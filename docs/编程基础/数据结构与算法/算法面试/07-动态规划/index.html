动态规划#

[toc]

算法思想#

什么是动态规划#

动态规划，英文：Dynamic Programming，简称DP，将问题分解为互相重叠的子问题，通过反复求解子问题来解决原问题就是动态规划，如果某一问题有很多重叠子问题，使用动态规划来解是比较有效的。

求解动态规划的核心问题是穷举，但是这类问题穷举有点特别，因为这类问题存在「重叠子问题」，如果暴力穷举的话效率会极其低下。动态规划问题一定会具备「最优子结构」，才能通过子问题的最值得到原问题的最值。另外，虽然动态规划的核心思想就是穷举求最值，但是问题可以千变万化，穷举所有可行解其实并不是一件容易的事，只有列出正确的「状态转移方程」才能正确地穷举。重叠子问题、最优子结构、状态转移方程就是动态规划三要素

动态规划和其他算法的区别#

  1. 动态规划和分治的区别：动态规划和分治都有最优子结构 ，但是分治的子问题不重叠。（因为有重复子问题，所以要搞记忆化搜索，而动态规划的dp数组就相当于记忆化！！最典型的是斐波那契数列，子问题有重复，如果不做记忆的话，会有大量的重复计算，加了记忆的话就是动态规划的思想了，动态规划也是一种空间换时间的思想！相对于普通的递归回溯效率要高。）（关于最优子结构的理解：大问题是由小问题推导而来，小问题有很多答案，我们要筛选出最优的那个，从而求出大问题的最优结果）
  2. 动态规划和贪心的区别：动态规划中每一个状态一定是由上一个状态推导出来的，这一点就区分于贪心，贪心没有状态推导，而是从局部直接选最优解，所以它永远是局部最优，但是全局的解不一定是最优的。
  3. 动态规划和递归的区别：递归和回溯可能存在非常多的重复计算，动态规划可以用递归加记忆化的方式减少不必要的重复计算。

参考：分治法，动态规划区别

动态规划的解题方法#

  • 递归+记忆化(自顶向下)
  • 动态规划（自底向上）

解动态规划题目的步骤#

  1. 根据重叠子问题定义状态
  2. 寻找最优子结构推导状态转移方程
  3. 确定dp初始状态
  4. 确定输出值

53. 最大子数组和-162#

给你一个整数数组 nums ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

子数组 是数组中的一个连续部分。

示例 1：

输入：nums = [-2,1,-3,4,-1,2,1,-5,4]
输出：6
解释：连续子数组 [4,-1,2,1] 的和最大，为 6 。

示例 2：

输入：nums = [1]
输出：1

示例 3：

输入：nums = [5,4,-1,7,8]
输出：23

来源：力扣（LeetCode） 链接：https://leetcode-cn.com/problems/maximum-subarray 著作权归领扣网络所有。商业转载请联系官方授权，非商业转载请注明出处。

特性：当前的最值可以由上一个子问题的最值推出来。

class Solution {
public:
int maxSubArray(vector<int>& nums) {
int n = nums.size();
if(n == 0) return 0;
//状态定义：dp[i] 表示以nums[i]为结尾的连续子数组的最大和
//初始状态：dp[i] = nums[i];
//状态转移：dp[i] = max(nums[i],dp[i-1]+nums[i])
//结果：dp数组的最大值
vector<int> dp(nums);//注意：用一个数组去初始化另一个数组！
for(int i = 1; i < n; i++) {
dp[i] = max(nums[i], nums[i]+dp[i-1]);
}
int ret = dp[0];
for(int i = 1; i < n; i++) {
ret = max(ret,dp[i]);
}
return ret;
}
};
class Solution {
public:
int maxSubArray(vector<int>& nums) {
int n = nums.size();
if(n == 0) return 0;
//状态定义：dp[i] 表示以nums[i]为结尾的连续子数组的最大和
//初始状态：dp[i] = nums[i];
//状态转移：dp[i] = max(nums[i],dp[i-1]+nums[i])
//结果：dp数组的最大值
//由于dp[i] 只和 dp[i-1]有关，所以可以进行状态压缩
int pre = nums[0], cur = pre, ret = pre;
for(int i = 1; i < n; i++) {
cur = max(nums[i], pre+nums[i]);
ret = max(ret, cur);
pre = cur;
}
return ret;
}
};

分析：这题让求出最大的子数组和。

子问题：是以num[i] 为结尾的最大子数组和。（一共有n个子问题）（也是状态定义）（重要：关于状态定义：要求最大子数组和，数组是[0..n-1]， 状态就定成最大子数组和！子问题就是：最大子数组和，数组是[0..n-2]，最大子数组和，数组是[0..n-3]，...最大子数组和，数组是[0..1]，最大子数组和，数组是[0],然后从小往大了推！！！）

最优子结构：num[i] 为结尾的子数组和有很多，我们选出最大的那个。

重叠子问题：计算dp[i]的时候，如果不记录dp[i-1]，name还需要重复计算dp[i-1],所有要记录下来！避免重复。

状态转移：才能通过子问题的最值得到原问题的最值。

121. 买卖股票的最佳时机 - 145#

分析

  1. 维度（维度是指影响最终结果的变量！）

每天的价格：天自然成为状态机的维度。

拥有股票/不拥有股票：第二个维度

交易次数：也可以成为一个维度。

  1. 状态转移

当前的状态是1：拥有股票

可有有两个状态转移过来：1. 前一天拥有股票，今天休息 2. 前一天不拥有股票，今天buy

当前的状态是0：不拥有股票

可有有两个状态转移过来：1. 前一天拥有股票，今天卖掉 2. 前一天不拥有股票，今天休息

dp[i][k][0 or 1] (0 <= i <= n-1, 1 <= k <= K)
  • i为天数
  • k为最多交易次数
  • [0,1]为是否持有股票
for 0 <= i <= n:
for 1 <= k <= K:
for s in {0,1}:
dp[i][k][s] = max(buy, sell,rest);

给定一个数组，它的第 i 个元素是一支给定股票第 i 天的价格。

如果你最多只允许完成一笔交易（即买入和卖出一支股票一次），设计一个算法来计算你所能获取的最大利润。

注意：你不能在买入股票前卖出股票。

示例 1:

输入: [7,1,5,3,6,4]
输出: 5
解释: 在第 2 天（股票价格 = 1）的时候买入，在第 5 天（股票价格 = 6）的时候卖出，最大利润 = 6-1 = 5 。
注意利润不能是 7-1 = 6, 因为卖出价格需要大于买入价格；同时，你不能在买入前卖出股票。

示例 2:

输入: [7,6,4,3,1]
输出: 0
解释: 在这种情况下, 没有交易完成, 所以最大利润为 0。

标准解法

class Solution {
public:
int maxProfit(vector<int>& prices) {
size_t n = prices.size();
/*
1.状态定义
dp[i][0] 第i天不持有股票的最大利润
dp[i][1] 第i天持有股票的最大利润
*/
vector<vector<int>>dp(n,vector<int>(2,0));
/*
2.状态初始化，递推的边界
*/
dp[0][0] = 0; //第0天不持有股票
dp[0][1] = -prices[0]; //第0天持有股票
/*
3.状态递推
设置初始值之后，循环体的开始是i=1
*/
for(int i = 1; i < n; i++) {
dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i]);//第i天不持有股票的最大利润 = max（前一天不持有股票的最大利润，前一天持有股票的最大利润+今天卖出的）
dp[i][1] = max(dp[i-1][1], -prices[i]);//第i天持有股票的最大利润 = max（前一天持有股票的最大利润，前一天不持有股票的最大利润-今天买入出的最大利润）前一天不持有股票的最大利润，在只能买卖一次的情况下是0
}
/*
4.确定输出值
最后一天不持有股票为最大利润
*/
return dp[n-1][0];
}
};

class Solution {
public:
int maxProfit(vector<int>& prices) {
size_t n = prices.size();
int sell = 0; //第0天不持有股票
int buy = -prices[0]; //第0天持有股票
//设置初始值之后，循环体的开始是i=1
for(int i = 1; i < n; i++) {
sell = max(sell, buy + prices[i]);//第i天不持有股票的最大利润 = max（前一天不持有股票的最大利润，前一天持有股票的最大利润+今天卖出的）

}
//最后一天不持有股票为最大利润
return sell;
}
};

class Solution:
import sys
def maxProfit(self, prices):
"""
:type prices: List[int]
:rtype: int
"""
#动态规划 前i天的最大收益 = max{前i-1天的最大收益，第i天的价格-前i-1天中的最小价格}
min_p = sys.maxsize
max_p = 0
n = len(prices)
for i in range(n):
min_p = min(min_p, prices[i])
max_p = max(max_p, prices[i] - min_p)
return max_p

42. 接雨水-93#

class Solution {
public:
int trap(vector<int>& height) {
//我们考察从第1列到height.size()-2列
int ret = 0;
//动态规划：当前的状态可由前面的状态递推而来
vector<int> max_left(height.size(), 0); //这个代表i左边的最大值的状态
vector<int> max_right(height.size(), 0);//这个代表i右边的最大值的状态
//左边最大值,状态数组
for(int i = 1; i < height.size()-1; i++) {
max_left[i] = max(max_left[i-1], height[i-1]);
}
//错误点：i--写成了i++
for(int i = height.size()-2; i > 0; i--) {
max_right[i] = max(max_right[i+1], height[i+1]);
}
for(int i = 1; i < height.size()-1; i++) {
//错误点：min重复
int min_two = min(max_left[i],max_right[i]);
if(min_two > height[i]) {
ret += (min_two - height[i]);
}
}
return ret;
}
};

class Solution {
public:
int trap(vector<int>& height) {
//按照层来求
int max_h = *(max_element(height.begin(), height.end()));
int ret = 0;
int temp_num = 0;
bool isStart = false;
for(int i = 1; i <= max_h; i++) {
isStart = false;
temp_num = 0;
for(int j = 0; j < height.size(); j++) {
//两高夹一低才能有水
//低位
if(isStart && height[j] < i) {
temp_num++;//开始之后，遇到低位temp++
}
//高位
if(height[j] >= i){//说明形成一个凹陷
isStart = true;
ret += temp_num;
temp_num = 0;//同时将统计归零
}
}
}
return ret;
}
};

class Solution {
public:
int trap(vector<int>& height) {
//按照列来求
int ret = 0;
for(int i = 1; i < height.size()-1; i++) {
//求其左边的最大值
int max_left = 0;
for(int j = 0; j < i; j++) {
max_left = max(max_left, height[j]);
}
//求其右边的最大值
int max_right = 0;
for(int j = i+1; j < height.size(); j++) {
max_right = max(max_right, height[j]);
}
//求两个最大值的最小值
int min_two = min(max_left, max_right);
//如果这个最小值大于当前列的话，那么当前列的值是最小值-当前列的值
if(min_two > height[i]) {
ret += (min_two - height[i]);
}
}
return ret;
}
};

300. 最长递增子序列 - 92#

定义状态：LIS(i)表示以nums[i]位结尾的最长上升子序列长度。

状态转移：LIS(i) = max(LIS(i), 1+LIS[j] ) 其中遍历j 从0 到i-1，且nums[i] > nums[j]。

初始状态：LIS(i) = 1; 从一个小的问题

确定输出值：LIS数组的最大值。

class Solution {
public:
int lengthOfLIS(vector<int>& nums) {
int n = nums.size();
if(n == 0) return 0;
//1.状态定义：dp[i]表示以nums[i]为结尾的最长上升子序列的长度
//2.初始状态：1
vector<int> dp(n, 1);
//3.状态转移
for(int i = 1; i < n; i++) {
//转移方程逻辑
for(int j = 0; j < i; j++) {
if(nums[i] > nums[j]) {
dp[i] = max(dp[i], 1+dp[j]);
}
}
}
//4.确定输出值
int ret = 1;
for(int i = 0; i < n; i++) {
ret = max(ret,dp[i]);
}
return ret;
}
};

  1. 状态定义

    dp[i] :爬上i阶台阶的方法

    dp[i] = dp[i-1]+dp[i-2] 爬一阶或者爬两阶过来

  3. 初始值

    dp[0] = 0,dp[1] = 1,dp[2] = 2;

  4. 输出值

    dp[n]

影响他的变量只有n，所以dp是一维的。表现出来是一层循环。

动态规划

class Solution {
public:
int climbStairs(int n) {
/*
1: 1
2: 2
3: 3
f(n) = f(n-1) + f(n-2);
*/
if(n < 3) {
return n;
}
//1.初始条件
int dp_pre1 = 2;
int dp_pre2 = 1;
//2.重复体结束条件
for(int i = 3; i <= n ; i++) {
//处理当前层逻辑
int dp = dp_pre1 + dp_pre2;
//为进入下一层做准备
dp_pre2 = dp_pre1;
dp_pre1 = dp;
}
return dp_pre1;
}
};

递归+记忆化搜索

递归是从后往前推。迭代是从前往后推。

class Solution {
private:
map<int, int> memo;
public:
int climbStairs(int n) {
return dfs(n);
}
int dfs(int level) {
//结束条件
if(level <= 2) {
return level;
}
//处理当前层逻辑(利用缓存进行剪枝，空间换时间)
if(memo[level] == 0) {
//drill down
memo[level] = dfs(level-1) + dfs(level-2);
}
return memo[level];
}
};

416. 分割等和子集-7#

状态转移方程：很多时候，状态转移方程思考的角度是「分类讨论」，对于「0-1 背包问题」而言就是「当前考虑到的数字选与不选」。

给你一个 只包含正整数 的 非空 数组 nums 。请你判断是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

示例 1：

输入：nums = [1,5,11,5]
输出：true
解释：数组可以分割成 [1, 5, 5] 和 [11] 。

示例 2：

输入：nums = [1,2,3,5]
输出：false
解释：数组不能分割成两个元素和相等的子集。

提示：

  • 1 <= nums.length <= 200
  • 1 <= nums[i] <= 100

来源：力扣（LeetCode） 链接：https://leetcode-cn.com/problems/partition-equal-subset-sum 著作权归领扣网络所有。商业转载请联系官方授权，非商业转载请注明出处。

/*
问题简析：从数组中选出一些值，使他们的和等于target。
1. 状态定义：
dp[i][j]表示从数组的 [0, i] 这个子区间内挑选一些正整数，每个数只能用一次，使得这些数的和恰好等于 j。
2. 状态转移方程：
很多时候，状态转移方程思考的角度是「分类讨论」，对于「0-1 背包问题」而言就是「当前考虑到的数字选与不选」。
- 不选择 nums[i]，如果在 [0, i - 1] 这个子区间内已经有一部分元素，使得它们的和为 j ，那么 dp[i][j] = true；
- 选择 nums[i]，如果在 [0, i - 1] 这个子区间内就得找到一部分元素，使得它们的和为 j - nums[i]。
状态转移方程：
dp[i][j] = dp[i - 1][j] or dp[i - 1][j - nums[i]]
3. 初始化：
dp[0][0] = false;因为候选数 nums[0] 是正整数，凑不出和为 0；
4. 确定输出值：
dp[len-1][target] 这里 len 表示数组的长度，target 是数组的元素之和（必须是偶数）的一半。
*/
class Solution {
private:
vector<vector<int>> memo;
//变量是index和sum,index和sum可能会被我们重复计算，所以要记忆化搜索。按照变化的参数的个数定义记忆化数组的维度和大小！
bool dfs(vector<int>&nums, int index, int sum) {
/*
1. 递归结束条件
*/
if(sum == 0) {
return true;
}
if(sum < 0 || index >= nums.size()) {
return false;
}
if(memo[index][sum] != -1) {
return memo[index][sum];
}
/*
2. 处理当前层逻辑
*/
memo[index][sum] = dfs(nums, index + 1, sum) || dfs(nums, index+1, sum - nums[index]);
return memo[index][sum];

class Solution {
public:
bool canPartition(vector<int>& nums) {
/*
从数组中选取一些元素，让他们的和等于 数组和的一半。
难点：对问题的抽象和转化。
*/
int n = nums.size();
if(n <= 1) return false;
int sum = 0;
for(int i = 0; i < n; i++) {
sum += nums[i];
}
if(sum&1) {//sum是个奇数
return false;
}
int target = sum>>1;
/*
1. 状态数组：行数i是物品索引，列数：容量
*/
vector<vector<bool>> dp(n,vector<bool>(target+1, false));
//2. 初始化
dp[0][0] = false;
if(nums[0] <= target) {
dp[0][nums[0]] = true;
}
//3. 状态转移
for(int i = 1; i < n; i++) { //物品一个一个的增加
for(int j = 0; j <= target; j++) { //容量一点一点的考虑
//当前值等于target表示肯定能找到何为j
if(nums[i] == j) {
dp[i][j] = true;
}
//当前容量小于target=j，可以选他，也可以不选他（分类讨论，状态转移）
else if(nums[i] < j) {
//1.不添加(容量是j)||添加(容量是j-nums[i]),添加之后容量得减少nums[i],j-nums[i] 是前n-1个值要凑出的target
dp[i][j] = dp[i-1][j] || dp[i-1][j-nums[i]];
}
//当前值大于target，肯定不能选他，继承上一个状态
else {
//不考虑当前元素的话，当前的状态就是i-1的状态。
dp[i][j] = dp[i-1][j];
}
}
}
return dp[n-1][target];
}
};

class Solution {
public:
bool canPartition(vector<int>& nums) {
/*
从数组中选取一些元素，让他们的和等于 数组和的一半。

}
if(sum&1) {//sum是个奇数
return false;
}

/*
1. 状态数组：行数i是物品索引，列数：容量
*/
vector<bool> dp(target+1, false);
//2. 初始化
dp[0] = false;
if(nums[0] <= target) {
dp[nums[0]] = true;
}
/*
nums[i] <= j 不满足，可以马上退出当前循环，因为后面的 j 的值肯定越来越小，没有必要继续做判断，
*/
//3. 状态转移
for(int i = 1; i < n; i++) { //物品一个一个的增加
for(int j = target; j > nums[i]; j--) { //容量一点一点的考虑
if(dp[target]) return true;
dp[j] = dp[j] || dp[j-nums[i]]; //从后往前推，添加或者不添加
}
}
return dp[target];
}
};

物品一个一个选，容量也一点一点增加去考虑，这一点是「动态规划」的思想

状态转移方程：很多时候，状态转移方程思考的角度是「分类讨论」，对于「0-1 背包问题」而言就是「当前考虑到的数字选与不选」。

  • 不选择 nums[i]，如果在 [0, i - 1] 这个子区间内已经有一部分元素，使得它们的和为 j ，那么 dp[i][j] = true；
  • 选择 nums[i]，如果在 [0, i - 1] 这个子区间内就得找到一部分元素，使得它们的和为 j - nums[i]。

本题与 0-1 背包问题有一个很大的不同，即：

0-1 背包问题选取的物品的容积总量 不能超过 规定的总量； 本题选取的数字之和需要 恰好等于 规定的和的一半。

给定一个数组，它的第 i 个元素是一支给定股票第 i 天的价格。

设计一个算法来计算你所能获取的最大利润。你可以尽可能地完成更多的交易（多次买卖一支股票）。

注意：你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

示例 1:

输入: [7,1,5,3,6,4]
输出: 7

随后，在第 4 天（股票价格 = 3）的时候买入，在第 5 天（股票价格 = 6）的时候卖出, 这笔交易所能获得利润 = 6-3 = 3 。

示例 2:

输入: [1,2,3,4,5]
输出: 4
解释: 在第 1 天（股票价格 = 1）的时候买入，在第 5 天 （股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4 。
注意你不能在第 1 天和第 2 天接连购买股票，之后再将它们卖出。

输入: [7,6,4,3,1]

  • 0 <= prices[i] <= 10 ^ 4
class Solution:
def maxProfit(self, prices):
"""
:type prices: List[int]
:rtype: int
"""
"""
数学原理或常识：
1.有多个波峰和波谷。一个数学原理是波峰-波谷的累加，必定大于最大波峰减去最小波谷。

return max_profit

dp[i][j] = dp[i-1][j] + dp[i][j-1]
return dp[m-1][n-1]

14. 最长公共前缀#

编写一个函数来查找字符串数组中的最长公共前缀。

如果不存在公共前缀，返回空字符串 ""。

示例 1:

输入: ["flower","flow","flight"] 输出: "fl" 示例 2:

输入: ["dog","racecar","car"] 输出: "" 解释: 输入不存在公共前缀。 说明:

所有输入只包含小写字母 a-z 。

来源：力扣（LeetCode） 链接：https://leetcode-cn.com/problems/longest-common-prefix 著作权归领扣网络所有。商业转载请联系官方授权，非商业转载请注明出处。

class Solution:
def longestCommonPrefix(self, strs):
"""
:type strs: List[str]
:rtype: str
"""
#获取数组长度
n = len(strs)
#滤掉空串
if n == 0:
return ""
#以数组长度为行数的二维数组按照列来遍历直到找到不相同。
#遍历最小二维数组长度
minL = len(strs[0])
for i in range(n):
m = strs[i]
if minL > len(m):
minL = len(m)
if minL == 0:
return ""
#以列为维度来遍历
length = 0
for i in range(minL):
isContinu = 1
p = strs[0][i]
for j in range(1,n):
if p != strs[j][i]:
isContinu = 0
break
if isContinu == 0:
break
#length长度加1 标识真正的长度，如果在取值时+1 则会导致无匹配时长度为1
length = i+1
return strs[0][0:length]

总结#

状态压缩#

  1. 如果状态只和上一个状态有关，可以压缩为两个变量来滚动。例如：53.最大子数组和-161

    0-1背包状态压缩为两行。

  2. 如果状态和前面所有的状态有关，不可以压缩，需要用循环来处理。例如：300.最长递增子序列-92

DP的维度#

维度是影响最终结果的变量。

  1. 如果dp只有1维，那么只需要一层循环。例如：53.最大子数组和-161，300.最长递增子序列-92
  2. 如果dp是多维，那么需要多层循环。例如：121.股票买卖的最佳时机-145

记忆化搜索#

  1. 按照dfs函数的变化的参数，来定义记忆化数组，一个变量是一维，数组如果有两个变量，则需要二维数组，n个变量n维数组。例如：416.分割等和子集-7

    定义数组大小的时候一定要注意！如果是0-n那么数组的大小是n+1。

状态转移#

  1. 很多时候，状态转移方程思考的角度是「分类讨论」。例如416.分割等和子集-7。数字的选与不选。
