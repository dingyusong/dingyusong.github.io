数据结构与算法

title: 数据结构与算法 toc: true date: 2018-10-08 20:09:28 tags: [编程基础,算法] categories: [编程基础]

效率#

人类对效率的追求是数据结构和算法发展的驱动力。

最直接的代表就是排序。最直观的选择排序效率是O(n^2),工业级的快速排序O(n*(logn))。这个差别有多大呢？

logn和n的差距

lognn
n=16416相差4倍
n=1024101024相差100倍
n=100万20100万相差5万倍

如果你有百万级别的数据量，假设用快速排序需要1天的话，选择排序需要5万天！

数据结构和算法解决的是如何更省、更快地存储和处理数据的问题。

计算的本质#

计算机非常强大，同时也非常傻。

他只会：取指->译码->执行。开机之后就永不停歇的运转。

具体到指令级别可分为：顺序执行和跳转。

顺序执行：一步一步的走。

跳转：调到某个标签。（在语言层面表现为：分支，循环，递归）

核心思想-重复性

递归和循环（while-loop，for-loop）(循环也叫迭代)没有本质区别，在指令级别都是跳转到某个tag，重复执行一段代码。

循环有结束条件，这个和递归的层数是一样的。（递归的结束除了从层这个级别考虑，大部分情况下还要从条件的情况去考虑，这叫剪枝）

for(int level = 0, level< maxLevel; level ++)

while(level < maxLevel)

void func(int level, int maxLevel)

迭代在函数体内，在重复体内借助函数变量即可完成某些功能。

迭代是单独的函数，则需要借助函数外的变量来完成某些功能。

重复性有结束条件。

递归和迭代是一种编程方式。

分治，回溯（深度优先搜索），贪心，动态规划，这些是编程思想。（他们一般都可以使用递归或者迭代来实现）

广度优先搜索也使用迭代，但是需要借助队列这种数据结构。

本质：借助一定的数据结构，在重复的过程中，往前演进，并寻找答案。

深度优先搜索和广度优先搜索对应的后继至少为2个，也就是要有分叉（树，图，多维数组）。

数据结构和算法紧密相关，有些算法必须和特定的数据结构结合。有些数据结构天然支持某些数据结构特性。

复杂数据结构的底层是基础数据结构。最基础的数据结构是数组和链表。（内存连续和不连续）

分为数据结构和算法两部分。

数据结构指的是数据的组织结构，数据是如何组织在一起的，属于静态行为。

一个数据最多和两个数据发生关系，前面一个，后面一个则这种是线性结构（数据被像一条线一样串起来），例如数组，单链表，双链表，栈，队列。 只有一个前驱，有多个后继属于树结构。堆是一种特殊的树结构。

可能有多个前驱，也可能有多个后继属于图结构。

算法是对数据的处理，属于动态行为。

程序=数据结构+算法。

数据结构：线性结构，树形结构，堆结构，图结构。

算法：交换，排序，最短路径，拓扑排序

底层数据结构：数组，链表(单链表，双链表)，树(二叉搜索树)

高层数据结构：栈，队列，集合，映射

高层数据结构都是基于底层数据结构来实现的。

高层数据结构：

定义好这种数据结构的使用接口+这种数据结构本身所维持的一些性质，很方便的放到一些应用中。

  • 集合（set）

最大的特性是去重。（比如统计每天到店的新客户有多少人，老客户就被去重了；一本书的词汇量；客户统计）

二分搜索树，默认是不能存放重复元素的。

有序集合：基于搜索树的实现

无序集合：基于哈希表的实现

多重集合：可以有重复元素

  • 映射（字典，map）

底层可以有链表和树 来实现。节点值包括（K，V）

有序映射：基于搜索树的实现

无序映射：基于哈希表的实现

多重映射：可以有重复元素

底层结构-树的变种：堆，线段树，并查集

  • 优先队列
